# Development

This `.development` directory will guide how this project
operates. Much of the material will be linked, and the intent
is to provide references to relevant context rather than
full context.

Some conventions, remember these:

- Files with names which are ALL_CAPS.md provide metadata
and guiding principles. This information tends to carry
over across projects.

- The same file name in lower case, e.g., all_aps.md will
have instructions specific to this project.

- Commit messages are a Very Big Deal, and require balancing
a number of constraints, some of which may seem partially
conflicting.

- Before staging changes, run all the actions in the CI
pipeline. Typically this will be testing and linting,
but could be a lot more. This is a good thing to remember.
